## 一、竞品核心功能拆解

元记笔记的"思维增强系统"包含以下 4 大 AI 能力：

### 1. 内容润色 ✍️
**描述**：一键优化表达，让文字更清晰有力
**适用场景**：
- 笔记写完后，AI 帮你润色文字表达
- 日记记录后，优化语言流畅度
- 待办描述不够清晰时，AI 帮助重新表述

**思格可实现方案**：
- 笔记编辑页增加"✨ AI 润色"按钮
- 调用已配置的 AI API（DeepSeek/通义千问/智谱 GLM）
- 润色选项：更简洁 / 更正式 / 更生动 / 纠正语法

### 2. 多角度分析（思维棱镜）🔍
**描述**：从多个维度拆解你的想法
**元记称"7 个维度"，可能包括**：
- 逻辑性：论证是否严密？
- 完整性：是否有遗漏的角度？
- 可行性：实际操作中会遇到什么问题？
- 风险点：潜在的风险是什么？
- 创新点：有什么独特的见解？
- 情感倾向：是否受情绪影响判断？
- 行动建议：下一步该怎么做？

**思格可实现方案**：
- 笔记详情页增加"🔍 AI 分析"入口
- 对当前笔记内容进行多维度深度分析
- 以卡片形式展示各维度结果（类似思维导图）

### 3. 自动摘要 📋
**描述**：长文秒变要点，快速回顾核心内容
**适用场景**：
- 笔记内容很长，需要快速了解要点
- 复习之前的笔记，不想重读全文
- 日记回顾，快速浏览关键事件

**思格可实现方案**：
- 主页笔记列表自动生成摘要（替代简单的文本截断）
- 笔记详情页显示"AI 摘要"区域
- 支持一句话摘要 / 要点列表 / 关键词提取 三种模式

### 4. 知识关联推荐 🔗
**描述**：自动发现笔记间的隐藏联系
**原理**：
- 通过标签、关键词、语义相似度匹配相关笔记
- 推荐与当前笔记相关的历史笔记
- 形成知识网络，帮助发现隐藏关联

**思格可实现方案**：
- 笔记详情页底部"相关笔记"推荐区
- 基于标签共现 + 内容关键词匹配
- 未来可引入向量嵌入做语义搜索

## 三、技术实现要点

### 已有基础
- ✅ AI 服务层 (`src/services/ai.js`) — 已支持 DeepSeek/通义千问/智谱 GLM
- ✅ AI 配置 UI (Profile.jsx) — 已有完整的 API Key 配置界面
- ✅ AI 测试连接功能 — 已实现

### 需要新增
1. **`src/services/ai-enhance.js`** — AI 增强服务模块
   - `summarize(content, mode)` — 摘要（一句话/要点/关键词）
   - `polish(content, style)` — 润色（简洁/正式/生动）
   - `analyze(content)` — 多维度分析
   - `findRelated(noteId, allNotes)` — 关联推荐

2. **UI 组件**：
   - 笔记编辑页：添加 "AI 涛色" 按钮 + 结果预览
   - 笔记详情页：添加 "AI 分析" 面板
   - 笔记列表：NoteCard 显示 AI 摘要替代截断文本

3. **性能考虑**：
   - AI 调用是异步的，需要 loading 状态
   - 摘要可以缓存（内容不变则不重复调用）
   - 长笔记可以先截取前 2000 字再调 API

## 四、Prompt 设计参考

### 摘要 Prompt
```
请对以下笔记内容进行摘要。要求：
1. 用一句话概括核心内容（不超过30字）
2. 提取3-5个关键要点（每条不超过20字）
3. 提取5个关键词

笔记内容：
{content}
```

### 润色 Prompt
```
请润色以下文字，使其更加{style}。
要求：
1. 保持原意不变
2. 优化语言表达
3. 不添加原文没有的信息

原文：
{content}
```

### 思维分析 Prompt
```
请从以下7个维度分析这段思考内容，每个维度给出简短评价和建议：
1. 逻辑性：论证是否严谨？有无逻辑漏洞？
2. 完整性：是否有遗漏的重要角度？
3. 可行性：实际执行中可能遇到什么困难？
4. 风险点：潜在的负面影响或风险？
5. 创新点：有哪些独特或有价值的见解？
6. 偏见检查：是否存在认知偏差或情绪影响？
7. 下一步行动：具体建议做什么？

内容：
{content}
```

## 五、用户原话

> "比如他说的思维增强系统，这是我真真正正想要的"

用户的核心诉求不是"又一个 AI 功能"，而是**真正能帮助思考和整理的工具**。这意味着：

1. **不要花哨**：功能要实用，不能只是调个 API 显示结果
2. **要融入工作流**：AI 增强应该在用户自然使用过程中出现，而不是单独的一个页面
3. **要有差异化**：不是简单的 ChatGPT 包装，而是针对"个人笔记/日记/思考"场景特化
